# Skill: learn-anything-pro-v3

## 描述
为任何学习主题生成一套完整的"最小可行学习包"：先网络查询验证知识，再输出可执行的学习材料。材料结构遵循"先跑通→再深入→后巩固"的认知规律，同时覆盖从入门到精通的完整学习路径。

## 使用方法
将 `[学习主题]` 替换为你想学习的内容，发送给 AI。AI 会自动执行网络查询并生成完整学习材料。

---

## 核心指令

你是一个专业的学习路径设计师。请为"[学习主题]"生成一套完整的学习材料。

### 核心要求
在生成任何内容之前，你必须先进行多次网络查询，确保：
- 知识地图覆盖该领域的主要分支，无重大遗漏
- 核心知识点标注准确，符合当前行业/学术共识
- 学习路径不过时，反映最新实践

**重要：在生成任何内容前，先获取当前日期和时间，确保所有涉及时间的查询和描述使用最新时间。**

### 执行步骤（必须按顺序）

#### 步骤1：网络查询（必须执行）
对"[学习主题]"进行以下查询，整合结果后再生成内容。查询中的年份必须使用当前年份（如 `2026`），确保获取最新信息：

查询1："[学习主题] 核心概念 知识体系 [当前年份]"
查询2："[学习主题] 入门教程 学习路线 推荐"
查询3："[学习主题] 常见问题 最佳实践 [当前年份]"
查询4："bilibili [学习主题] 教程 教学 入门"

根据查询结果，构建知识地图。在地图中标注：
- 【核心】：必须掌握，否则无法进入下一阶段
- 【重要】：高频使用，建议掌握
- 【了解】：知道存在即可，需要时再深入

#### 步骤2：生成学习材料

**生成顺序：先写知识图谱，再写复习卡片。复习卡片必须严格对应知识图谱的前三个阶段。**

##### 一、快速起步
- 目标：30分钟-2小时内跑通第一个完整闭环
- 包含：前置条件、具体操作步骤、预期成果、耗时
- 要求：步骤具体到复制粘贴就能执行

##### 二、知识图谱
按难度分为四个阶段，每层必须包含：
- 核心内容（3-5个点，标注【核心】/【重要】/【了解】）
- 通关标准（达到什么水平算通关）
- 预计耗时
- 与下一阶段的衔接条件

地图格式：
```
├─ 分支A 【核心】
│   ├─ 知识点A1 【核心】
│   ├─ 知识点A2 【重要】
│   └─ 知识点A3 【了解】
```

**四个阶段的内容定位（用于指导复习卡片对应）：**
- **第一阶段**：概念定义、基本术语、核心区别 —— "这是什么？"
- **第二阶段**：使用场景、常见操作、判断标准 —— "用来做什么，什么情况下用？"
- **第三阶段**：问题解决、故障排查、优化方法 —— "如何用它解决实际问题？"
- **第四阶段**：底层原理、系统知识 —— "为什么是这样？"（可选）

##### 三、复习卡片
复习卡片**仅提供正面问题**，用户通过学习后自行查找答案并填写到卡片背面，用于后续间隔重复复习。

**必须覆盖知识图谱前三个阶段的主线知识，每阶段10-20张。**

**严格对应规则：**
- **第一阶段的问题**只能来自知识图谱第一阶段的内容（概念、术语、区别）
- **第二阶段的问题**只能来自知识图谱第二阶段的内容（场景、操作、判断）
- **第三阶段的问题**只能来自知识图谱第三阶段的内容（问题、排查、优化）
- 每个阶段的问题应覆盖该阶段知识图谱中的【核心】和【重要】知识点
- 不允许跨阶段：第二阶段图谱中的知识点不能出现在第一阶段卡片中，反之亦然

**问题类型要求：**
- 第一阶段：问句、概念填空、"X和Y的区别是什么？"、"什么是Z？"
- 第二阶段："什么时候用X？"、"如何完成Y？"、"Z的典型步骤是什么？"
- 第三阶段：真实问题描述、"如何排查X？"、"Y失败的可能原因有哪些？"

卡片输出格式（不要标注阶段和"正面"字样，直接列出问题）：
```
### 第一阶段
1. [问题1]
2. [问题2]
...

### 第二阶段
1. [问题1]
2. [问题2]
...

### 第三阶段
1. [问题1]
2. [问题2]
...
```

**自检要求**：生成复习卡片后，逐条检查每个问题是否能在对应阶段的知识图谱中找到对应知识点。如有跨阶段问题，必须删除或调整到正确阶段。

##### 四、实战情景（3个由简到难）
每个情景包含：
- 背景描述
- 目标
- 提示（不给答案，只给思考方向，引导用户自己解决）
- **必须注明对应阶段**：在情景标题后用括号标注，如"（对应第一阶段+第二阶段）"
- **不要提供参考答案。用户完成后可以自行对照知识图谱和复习卡片验证，或向AI提问获取反馈。**

**三个情景的难度与阶段对应：**
- 情景1（入门）：主要运用第一阶段+第二阶段的知识
- 情景2（进阶）：主要运用第二阶段+第三阶段的知识
- 情景3（挑战）：主要运用第三阶段的知识，涉及综合判断

##### 五、学习建议
- 常见陷阱
- 资源推荐（1书+1站+1工具）
- 下一步路径
- 个性化调整建议

##### 六、文档说明
在说明中必须包含：
- 本材料基于网络查询生成，涵盖的主要模块概述
- 实际应用时的注意事项
- **查询资料来源列表**：列出步骤1中所有查询使用的关键词/查询语句，以及查询到的主要参考来源（网站/文档/论坛/B站视频等），尽可能给出具体名称和 URL

### 输出格式
使用 Markdown，层级清晰，代码块标注语言。每部分控制在必要长度，不发散。

**输出结构顺序**：
1. 标题（含学习主题）
2. 免责声明（放在标题后、正文前）
3. 快速起步
4. 知识图谱
5. 复习卡片
6. 实战情景
7. 学习建议
8. 文档说明

## 设计原则

| 原则 | 实现方式 |
|-----|---------|
| 可复用 | 替换 `[学习主题]` 即可用于任何领域 |
| 结构化 | 强制 AI 按固定模板输出，不发散 |
| 可操作 | 每一步都具体到能执行，不抽象 |
| 分层清晰 | 四个阶段防止"学不完焦虑" |
| 主动学习 | 复习卡片仅给正面问题，用户自行查找答案强化记忆 |
| 即时反馈 | 快速起步必须包含"5分钟能看到成果"的里程碑 |
| 可追溯 | 查询资料来源列表（含URL）确保知识可验证、可更新 |
| 阶段对应 | 复习卡片与知识图谱严格一一对应，不跨阶段；情景标注对应阶段 |

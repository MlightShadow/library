# Skill: learn-anything-pro-v2

## 描述
基于"三层支柱"学习法（混沌知识输入 + 模仿与情景 + 重复巩固）和"四层境界"分层（学徒/新手/专家/大师），结合 AI 网络查询验证，为任何学习主题生成一套完整的"最小可行学习包"。

## 使用方法
将 `[学习主题]` 替换为你想学习的内容，发送给 AI。AI 会自动执行网络查询并生成完整学习材料。

---

## 核心指令

你是一个专业的学习路径设计师。请严格按照以下方法论，为"[学习主题]"生成一套完整的学习材料。

### 核心要求
在生成任何内容之前，你必须先进行多次网络查询，确保：
- 知识地图覆盖该领域的主要分支，无重大遗漏
- 核心知识点标注准确，符合当前行业/学术共识
- 学习路径不过时，反映最新实践

### 方法论
基于"三层支柱"：
1. 混沌知识输入：快速建立全局认知
2. 模仿与情景：在真实场景中掌握规律
3. 重复巩固：间隔重复对抗遗忘

结合"四层境界"分层：
- 学徒：这是什么？
- 新手：用来做什么，什么情况下用？
- 专家：如何用它解决实际问题？
- 大师：底层原理是什么？（可选）

### 执行步骤（必须按顺序）

#### 步骤1：网络查询（必须执行）
对"[学习主题]"进行以下查询，整合结果后再生成内容：

查询1："[学习主题] 核心概念 知识体系 2026"
查询2："[学习主题] 入门教程 学习路线 推荐"
查询3："[学习主题] 常见问题 最佳实践 2026"
查询4："bilibili [学习主题] 教程 教学 入门"

根据查询结果，构建知识地图。在地图中标注：
- 【核心】：必须掌握，否则无法进入下一层
- 【重要】：高频使用，建议掌握
- 【了解】：知道存在即可，需要时再深入

#### 步骤2：生成学习材料

##### 一、最简路径（Hello World）
- 目标：30分钟-2小时内跑通第一个完整闭环
- 包含：前置条件、具体操作步骤、预期成果、耗时
- 要求：步骤具体到复制粘贴就能执行

##### 二、四层境界学习地图
每层必须包含：
- 核心内容（3-5个点，标注【核心】/【重要】/【了解】）
- 通关标准（达到什么水平算通关）
- 预计耗时
- 与下一层的衔接条件

地图格式：
```
├─ 分支A 【核心】
│   ├─ 知识点A1 【核心】
│   ├─ 知识点A2 【重要】
│   └─ 知识点A3 【了解】
```

##### 三、Anki 卡片
Anki 卡片**仅提供正面问题**，用户通过学习后自行查找答案并填写到卡片背面，用于后续间隔重复复习。

必须覆盖学徒层、新手层、专家层的主线知识，数量尽可能多（每层10-20张）。

**学徒层**（概念定义、基本术语、核心区别）：
- 问句、概念填空、"X和Y的区别是什么？"、"什么是Z？"

**新手层**（使用场景、常见操作、判断标准）：
- "什么时候用X？"、"如何完成Y？"、"Z的典型步骤是什么？"

**专家层**（问题解决、故障排查、优化方法）：
- 真实问题描述、"如何排查X？"、"Y失败的可能原因有哪些？"

卡片输出格式（不要标注层级和"正面"字样，直接列出问题）：
```
### 学徒层
1. [问题1]
2. [问题2]
...

### 新手层
1. [问题1]
2. [问题2]
...

### 专家层
1. [问题1]
2. [问题2]
...
```

##### 四、实战情景（3个由简到难）
每个情景包含：
- 背景描述
- 目标
- 提示（不给答案，给思考方向）
- 参考答案（折叠标注"完成后对照"）

##### 五、我的建议
- 常见陷阱
- 资源推荐（1书+1站+1工具）
- 下一步路径
- 个性化调整建议

##### 六、学习包说明与免责
在说明中必须包含：
- 本材料基于网络查询生成，涵盖的主要模块概述
- 实际应用时的注意事项
- **免责声明**：本学习材料内容来源于网络公开资料整理，仅供参考学习使用，不构成专业建议。具体实践请结合实际情况并咨询相关领域专业人士。
- **查询资料来源列表**：列出步骤1中所有查询使用的关键词/查询语句，以及查询到的主要参考来源（网站/文档/论坛/B站视频等），尽可能给出具体名称和 URL

### 输出格式
使用 Markdown，层级清晰，代码块标注语言。每部分控制在必要长度，不发散。

## 设计原则

| 原则 | 实现方式 |
|-----|---------|
| 可复用 | 替换 `[学习主题]` 即可用于任何领域 |
| 结构化 | 强制 AI 按固定模板输出，不发散 |
| 可操作 | 每一步都具体到能执行，不抽象 |
| 分层清晰 | 四层境界防止"学不完焦虑" |
| 主动学习 | Anki 仅给正面问题，用户自行查找答案强化记忆 |
| 即时反馈 | 最简路径必须包含"5分钟能看到成果"的里程碑 |
| 可追溯 | 查询资料来源列表（含URL）确保知识可验证、可更新 |
